🔢 RPN vs ALG Mode

Understanding how to enter numbers and operations is essential for efficient calculation.

Reverse Polish Notation (RPN)

In RPN mode, you enter the operands first, then press the operator. The calculator uses a 4-level stack (X, Y, Z, T) to hold intermediate values.

💡 Rule: To compute A + B, press: A ENTER B +

RPN Stack Concept

The stack automatically lifts when you start a new number. Think of it as a stack of plates:

  1. Type a number — it goes into the X register (bottom)
  2. Press ENTER — X lifts to Y, number stays in X
  3. Type second number — goes into X
  4. Press operator (+, −, ×, ÷) — result appears in X
📝 Example: Calculate (3 + 4) × 2 in RPN

The most powerful feature of the HP 12C — solving for any TVM variable when you know the other four.

⚠️ Cash Flow Sign Convention: Money paid out (outflows) is negative. Money received (inflows) is positive. Use CHS to change sign.

General TVM Procedure

  1. Press f CLEAR FIN — clears all financial registers
  2. Enter the known values: type the number, then press the register key (n, i, PV, PMT, or FV)
  3. If rate is annual and periods are monthly, divide rate by 12: 6 g 12÷
  4. If periods are years×months: 5 g 12× for 5 years monthly
  5. Press the unknown register key to calculate it

💹 Interest Calculations

Simple interest, compound interest, and percentage calculations.

Simple Interest

The HP 12C calculates simple interest on both a 360-day and 365-day basis simultaneously.

Simple Interest = Principal × Annual Rate × (Days / Day-Basis)
  1. Key in number of days, press n
  2. Key in annual interest rate (%), press i
  3. Key in principal amount, press CHS PV
  4. Press f INT → shows interest on 360-day basis
  5. For 365-day basis: press RCL × (or use d key to toggle)
  6. To get total (principal + interest): press +

📝 Example: Investment of $10,000. Cash flows over 4 years: $2,000, $4,000, $4,000, $2,000. Is it profitable at 12% discount rate?
KeystrokeDisplayAction

Interpretation: A negative NPV means the investment doesn't meet the 12% hurdle rate. The IRR of 10.28% confirms the return is below the 12% required.

NPV vs IRR Interpretation

NPV Decision Rule
NPV > 0✅ Accept — adds value
NPV = 0⚖️ Indifferent
NPV < 0❌ Reject — destroys value
IRR Decision Rule
IRR > hurdle rate✅ Accept investment
IRR = hurdle rate⚖️ Indifferent
IRR < hurdle rate❌ Reject investment
📉 Depreciation

Calculate depreciation using three different methods — Straight-Line, Declining Balance, and Sum-of-Years' Digits.

Setup for All Depreciation Methods

  1. Enter original cost → PV
  2. Enter salvage value → FV
  3. Enter useful life in years → n
  4. Enter the year number for which to calculate → i (for SL/SOYD) or as multiplier (for DB)

Straight-Line Depreciation

Annual Depreciation = (Cost − Salvage) ÷ Useful Life

Keystroke: j (Year number) i then f SL

💾 Why Use Programs?

Programs let you store a sequence of keystrokes and replay them instantly — press one key instead of many.

💡

What is a Program?

A program is simply a sequence of keystrokes stored inside the calculator. Instead of pressing all the keys each time, you press just one key — R/S — to start the program. The calculator does the rest automatically, using whatever data you enter beforehand.

Two Operating Modes

▶ RUN Mode Default

Keys are executed immediately when pressed. This is normal calculator operation.

f RUN → switches to Run mode
PRGM indicator turns OFF
📝 PRGM Mode

Keys are stored as instructions — not executed. Use this to write programs.

f P/R → switches to Program mode
PRGM indicator turns ON

Program Memory

The HP 12C Platinum stores up to 400 program lines. Each line holds one instruction (one keystroke). Program memory and data storage registers share memory — the more program lines used, the fewer data registers are available, and vice versa.

Default StateValue
Program lines available8 (auto-expands as needed)
Data registers available20 (R₀ through R₉, R.0–R.9)
Maximum program lines400
Check memory status: Press g MEM — display shows available program lines and registers.

The Program Workflow

  1. Plan it first — write down the exact keystrokes on paper before entering them
  2. Enter Program mode — press f P/R (PRGM indicator lights up)
  3. Clear old programs — press f CLEAR PRGM (shows 000,)
  4. Enter keystrokes — each key press stores one instruction; display shows line number and code
  5. Return to Run mode — press f P/R again
  6. Key in data — enter the input values your program needs
  7. Run it — press R/S and the program executes automatically

Stepping Through a Program (Debugging)

Instead of pressing R/S to run the entire program, use SST to execute one line at a time. This lets you see each intermediate result and find errors.

Pausing During Execution

Add a g PSE (Pause) instruction inside a program to briefly display an intermediate result (~1 second) before continuing. Useful for showing multiple outputs from one program run.

To stop during a pause: Press any key while the result is showing. Press R/S to resume execution from that point.
🔁 Branching & Looping

Transfer execution to different program lines and repeat sections automatically.

Simple Branching — GTO (Go To)

The GTO instruction tells the calculator to jump to a specific program line instead of continuing to the next line.

KeystrokeWhen Used In a Program
g GTO 005Jumps execution to program line 005
g GTO 000Returns to line 000 (halts program — auto-stored at end)
Direction matters: GTO can branch backward (creates loops) or forward (skips sections). Backward branching without a stop condition creates an infinite loop!

Looping

A loop is created by branching backward — placing a GTO instruction that points to an earlier line. The program will keep repeating that section until you stop it or a conditional branch exits the loop.

Conditional Branching

Conditional branches let the program make decisions — going to one line if a condition is true, and simply continuing to the next line if false.

🛠️ Editing Programs

How to change, add, or delete instructions after a program is stored.

Viewing the Program

  1. Press f P/R to enter Program mode (PRGM indicator on)
  2. Press g GTO . 000 to go to line 000
  3. Press SST repeatedly to step through each line and see the keycode stored there

Changing an Instruction

To replace the instruction on a specific line with a new one:

  1. Enter Program mode (f P/R)
  2. Navigate to one line before the line you want to change using SST or g GTO NNN
  3. Press SST one more time to arrive at the target line
  4. Simply press the new key — it overwrites the existing instruction
⚠️ There is no "insert" in simple overwrite mode. Pressing a key at a line replaces whatever was there before.

Adding Instructions at the End

  1. Enter Program mode
  2. Step through to the last instruction of your existing program using SST
  3. Press the new keystroke(s) — they will be added after the last instruction

Adding Instructions Within a Program

Because there is no true "insert" function, there are two techniques:

Best for short insertions. Re-key the affected lines by overwriting them with the new content + the instructions you're displacing.

  1. Navigate to the line where you want to insert
  2. Type the new instruction(s) — they overwrite from this line forward
  3. Continue typing the original instructions that were "pushed down" so they follow correctly
Example: If you have lines 5, 6, 7 and want to insert a new step before line 6, go to line 5, then type: [new instruction] [old line 6] [old line 7].

Best for larger insertions. Use GTO to detour the program to free space at the end of memory for the new instructions.

  1. Replace the line before the insertion point with a g GTO NNN instruction pointing to a free area at the end of program memory
  2. At that free area, key in the new instructions
  3. Then key in the original instructions that were "skipped" by the GTO
  4. End with a g GTO back to the next line in the original program

Multiple Programs

You can store more than one program in memory at the same time. Each program starts at a specific line number. Use g GTO NNN to start execution at a specific program's first line.

💡 Tips & Tricks

Power-user techniques to get more out of your HP 12C Platinum.

Top 10 Tips for New Users

  1. Always clear before a new calculation. Press f CLEAR FIN before every new TVM problem to avoid using stale values.
  2. Use the shortcut keys for monthly calculations. g 12× (years → months) and g 12÷ (annual rate → monthly rate) save time.
  3. The sign convention is your friend. If you get a wrong answer, check the signs. Outflows are negative — use CHS to flip.
  4. Recall without recalculating. Use RCL + key to see a stored value without triggering a calculation.
  5. x↔y swaps the two registers. Very useful after AMORT to see both interest and principal portions.
  6. LAST X recovers your last operand. Press g LAST X to recover the number used in the most recent operation — handy for error recovery.
  7. Continuous Memory keeps your data. Turning the calculator off doesn't erase anything — all registers are preserved until you explicitly clear them.
  8. Display format affects only what you see. The calculator always uses full 10-digit precision internally, regardless of how many decimals are displayed.
  9. AMORT automatically updates the register. After running AMORT for period 1, pressing AMORT again gives period 2 — you don't need to re-enter n.
  10. BEG vs END matters greatly. Lease and annuity-due problems use BEG; standard loan payments use END. Getting this wrong changes the answer.

Date Calculations

The HP 12C can calculate the number of days between two dates, or find a future/past date.

  1. Set date format: g D.MY (day/month/year) or g M.DY (default: month/day/year)
  2. Enter first date as MM.DDYYYY, press ENTER
  3. Enter second date as MM.DDYYYY
  4. Press g DAYS — shows actual days between dates
  5. Press x↔y to see days on 360-day basis
